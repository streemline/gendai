Техническое задание на разработку приложения для учета рабочего времени

Общая информация

Приложение предназначено для учета рабочего времени сотрудников на Android. 
Оно будет включать в себя функционал для ввода данных, их сохранения, вычислений, экспорта и отображения статистики. 

Функциональные требования

Интерфейс ввода данных:

Главный экран содержит форму ввода данных, повторяющую структуру таблицы, включая следующие поля:

Дата
Название и место мероприятия
Описание деятельности
Время начала и окончания работы
Перерыв в рабочем времени
Количество часов
Ставка (Крон/час)
Общая сумма за день
Подпись сотрудника
В форме предусмотрены текстовые поля, элементы выбора времени и область для цифровой подписи.

Сохранение данных:
Информация сохраняется в локальной базе данных.

Расчеты:
Автоматический подсчет:
Общего количества часов: разница между временем начала и окончания работы за вычетом перерывов.
Итоговой суммы за день: умножение общего количества часов на ставку.

Экспорт данных:
Экспорт в форматы Excel и PDF:
Экспортируемые файлы включают все данные, введенные пользователем.

Экран статистики:
Отображение общей информации:
Общее количество отработанных часов.
Итоговая сумма оплаты за месяц.
Детализация по дням с использованием графиков и таблиц.

Цифровая подпись:
Подпись через сенсорный экран сохраняется как изображение и сохраняется в базе данных.

Локализация:
Приложение поддерживает три языка:
Русский
Украинский
Чешский

Адаптивный дизайн:
Использование технологий адаптивного дизайна для работы на экранах различных размеров.

Основные элементы интерфейса:.
Поддержка светлой темы с акцентным синим цветом для интерактивных элементов.

Дополнительные требования
Модели данных предоставляют структуры для хранения информации.

Документация:
Подробные комментарии в коде, объясняющие назначение классов и методов.
README-файл с инструкциями по установке и запуску приложения.

Инструкции по запуску:
Установка необходимых зависимостей.
Настройка локальной базы данных.
Примеры использования интерфейса приложения.

Дизайн интерфейса
Тема: Светлая тема для обеспечения читаемости.
Цветовая схема: Основной цвет — синий (для кнопок и интерактивных элементов).
Кнопки: Закругленные углы (радиус 8dp) для современного вида.

Результат
Приложение будет:

Удобным в использовании благодаря продуманному интерфейсу.
Функциональным за счет автоматизации расчётов и экспорта данных.
Легким в локализации и адаптации под разные устройства.
